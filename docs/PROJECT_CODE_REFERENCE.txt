Shared project patterns
-----------------------
- Most pages use the same shell: skip link, sticky header, nav, main content, footer, install button, and back-to-top button.
- Most pages load the same shared styles: defs.css, base.css, components.css, layout.css, and pages.css.
- Most pages also load the same shared scripts: navigation.js, modals.js, search-filter.js, forms.js, and main.js.
- Organization listing pages use .org-card elements with data-category, data-logo, data-focus, data-impact, and data-website attributes.
- User data is stored locally in localStorage. Important keys include:
  - ecoTasks
  - feedbackList
  - ecoTrees
  - ecoPlastic
  - ecosphere-installed
- The site is a PWA. manifest.json describes install behavior and js/service-worker.js provides offline caching.

HTML tag glossary
-----------------
- <html>: Root document element. Wraps the entire page and sets the language with lang="en".
- <head>: Stores metadata, linked assets, and page setup information.
- <meta>: Defines document encoding, viewport, theme color, descriptions, social sharing metadata, and PWA hints.
- <title>: Sets the browser tab title and search-result title text.
- <link>: Connects external resources such as stylesheets, manifest.json, canonical URLs, preload, and prefetch assets.
- <script>: Loads JavaScript files or embeds inline JSON-LD structured data.
- <body>: Contains all visible page content.
- <a>: Creates links for navigation, CTA buttons, skip links, legal links, and external references.
- <header>: Wraps the top navigation area.
- <nav>: Groups navigation links for primary nav, footer nav, legal nav, and social nav.
- <main>: Marks the main page content for accessibility and structure.
- <section>: Breaks content into large semantic blocks like hero, cards, policy, and CTA sections.
- <div>: Generic layout container used throughout the project.
- <h1>: Primary page title.
- <h2>: Section heading, CTA heading, or policy subsection heading.
- <h3>: Card title, footer brand label, or content heading.
- <h4>: Footer subsection heading.
- <p>: Body copy, descriptions, helper text, and status text.
- <span>: Small inline label, badge, icon text, or helper content.
- <strong>: Adds semantic emphasis for important inline text.
- <button>: Used for nav toggle, install button, filter buttons, modal close, form submit, share buttons, and retry actions.
- <input>: Text and email inputs for search and forms.
- <label>: Accessible label for form fields and search inputs.
- <textarea>: Multi-line feedback input.
- <form>: Wraps the feedback form.
- <img>: Displays media cards and hero portraits.
- <svg>: Inline vector graphic used for the home-page brand mark.
- <rect>: Rectangle shape inside the inline SVG logo.
- <path>: Path shape inside the inline SVG logo.
- <circle>: Circle shape inside the inline SVG logo.
- <ul>: Unordered list used in the license page.
- <li>: List item used in the license page.
- <style>: Inline CSS block used only in offline.html.
- <br>: Forced line break used in offline.html.

3. js/modals.js
Purpose:
- Reusable modal system for organization cards and progress popups.
Functions and state:
- lastFocusedElement: stores the last focused element so focus can be restored after closing.
- window.showGenericModal(title, contentHtml): generic modal renderer used by the tracker progress popups.
- getEmojiLogo(title): maps an organization name or keyword to an emoji fallback logo.
- renderOrgLogo(container, fallbackText): builds the logo block used in cards and modal headers.
- initializeModals(): sets ARIA attributes, wires close behavior, decorates .org-card elements, and binds click/keyboard open handlers.
- handleModalKeydown(event): handles Escape to close and Tab focus trapping.
- openModal(card): reads a clicked card's data-* attributes and builds the modal content.
- closeModal(): hides the modal, unlocks page scroll, and restores focus.
What it does:
- Turns static organization cards into accessible modal launchers.
- Reuses the same modal shell for the task-page analysis popup.

6. js/main.js
Purpose:
- Main application bootstrap, PWA install flow, tracker generation, live counters, and service worker registration.
Important variables/constants:
- deferredPrompt: stores the beforeinstallprompt event for later use.
- DEBUG_MODE: enables console logging on localhost or with ?debug=true.
- INSTALL_STATE_KEY: localStorage key for install state.
- ADD_TO_HOME_LABEL and REMOVE_FROM_HOME_LABEL: button labels for install state.
- performanceMetrics: simple in-memory performance markers.
Install and PWA helper functions:
- debugLog(method, ...args): guarded console logger for local debugging.
- logTime(label): helper around performance.now().
- isStandaloneMode(): checks if the app is running like an installed app.
- isIndexPage(): checks whether the current page is the landing page.
- getStoredInstallState(): reads install state from localStorage.
- setStoredInstallState(isInstalled): saves install state.
- isAppInstalled(): combines standalone-mode and stored install checks.
- removeInstallBanner(): removes the bottom install banner if present.
- setInstallButtonVisible(): updates the footer install button label.
- refreshInstallUi(): syncs install UI with current state.
- promptForInstall(): triggers the stored install prompt and updates state after the user's decision.
- showInstallPrompt(): creates the install banner with Install and Dismiss buttons.
- triggerInstall(): runs the install flow or shows a helpful alert if the app is already installed or the browser is not ready.
Animation and page feature functions:
- measurePerformance(label): stores a startup timing mark.
- initializeScrollAnimations(): uses IntersectionObserver to add reveal-visible when cards enter view.
- initializeSustainabilityTracker(): builds the task UI inside #ecoTracker, manages task completion, stores progress in localStorage, and opens analysis popups.
- startTreeCounter(): animates and persists the landing-page counters for trees and recycled plastic.
- initializeSocialPlaceholders(): blocks future social links and shows an alert explaining they are not active yet.
- window.handleWebShare(title, text, url): wrapper around navigator.share with fallback alert.
- reportMetrics(): logs readiness timing in debug mode.
Boot and performance functions:
- DOMContentLoaded handler: initializes shared features in a safe init() wrapper, registers the service worker, refreshes PWA UI, and starts feature modules.
- trackCoreWebVitals(): uses PerformanceObserver to log LCP and first-input metrics while debugging.
Service worker and update functions:
- registerServiceWorker(): registers ./service-worker.js, polls for updates, watches updatefound, and listens for service worker messages.
- showUpdatePrompt(): creates an update banner when a new service worker is installed.
- showUpdateNotification(): shows a temporary notification that offline content is ready.
What it does:
- Acts as the main coordinator for nearly every interactive feature on the site.

7. js/service-worker.js
Purpose:
- Offline caching and fetch handling for the PWA.
Important constants:
- CACHE_VERSION: versioned name for the app-shell cache.
- RUNTIME_CACHE: versioned name for runtime network responses.
- ASSETS_TO_CACHE: array of HTML, CSS, JS, manifest, and icon assets to cache during install.
Functions and event handlers:
- updateRuntimeCache(request, response): saves successful network responses in the runtime cache.
- self.addEventListener('install', ...): opens the main cache, adds the app shell assets, and calls skipWaiting().
- self.addEventListener('activate', ...): removes old cache versions and claims clients.
- self.addEventListener('fetch', ...): serves cached responses first, refreshes in the background, caches fresh network responses, and falls back to offline.html for failed documents.
- self.addEventListener('message', ...): allows SKIP_WAITING messages to activate a waiting worker immediately.
What it does:
- Makes the site available offline and keeps cached assets updated over time.

JSON reference
--------------
manifest.json
Purpose:
- Web app manifest for installability and app-like behavior.
Keys and meaning:
- name: Full installable app name shown by the browser or OS.
- short_name: Short label for home screen and launcher spaces.
- description: Summary of the app for install surfaces.
- start_url: Entry point after launch.
- scope: URL range controlled by the installed app.
- display: Uses standalone so the app opens without browser UI.
- orientation: Locks preferred orientation to portrait-primary.
- background_color: Splash/background color.
- theme_color: Browser UI accent color.
- icons[]: Install icons for 192x192 and 512x512 usage, including maskable support.
- screenshots[]: App screenshot metadata for install UI.
- categories[]: App categories for stores or install surfaces.
- shortcuts[]: Quick actions linking to Organizations, Feedback, and Tasks.
- prefer_related_applications: false, meaning the web app is preferred over suggesting native apps.
What it does:
- Tells the browser how EcoSphere should look and behave when installed as an app.

XML reference
-------------
sitemap.xml
Purpose:
- Search engine sitemap for SEO and crawl guidance.
Elements and meaning:
- <urlset>: Root sitemap container with sitemap, image, and mobile namespaces.
- <url>: One URL entry.
- <loc>: Canonical URL of a page.
- <lastmod>: Last modification date/time.
- <changefreq>: Expected update frequency.
- <priority>: Relative crawl priority.
- <mobile:mobile/>: Marks the page as mobile-compatible.
What it does:
- Lists the public pages that should be indexed by search engines.
- Includes the main landing page, organization pages, task page, feedback page, and legal pages.
- Does not include offline.html, which is correct because that page is a fallback document rather than a normal discovery page.

Important implementation notes
------------------------------
- Organization card modals are data-driven. The HTML cards provide the data and js/modals.js renders the popup.
- The task page is mostly generated by JavaScript. task.html only provides the target container and modal shell.
- Feedback is saved only on the user's device in localStorage in the current version.
- The PWA install flow is implemented, but share buttons on heroes.html are not currently connected to the share helper.
- The root service worker exists only to keep scope correct. The actual logic lives in js/service-worker.js.
